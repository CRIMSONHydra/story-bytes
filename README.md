# Story Bytes

Local-first toolkit for building a story knowledge base with chapter summaries, Q&A, and multimodal context.

## Tech Stack (local friendly)

| Concern            | Choice                                                      |
|--------------------|-------------------------------------------------------------|
| Backend API        | Express + TypeScript (Node.js 20, npm)                      |
| Frontend           | React 18 + TypeScript via Vite                              |
| Data Pipelines     | Python 3.12 (managed via `uv`)                              |
| Database           | PostgreSQL 18.1 (localhost) + `pgvector`                    |
| Storage            | Local filesystem/NAS (can be swapped for S3 later)          |
| Jobs / Queues      | Redis + BullMQ (Node) and/or Python workers (future)        |

## Repo Layout

```
backend/    Express API (TypeScript)
frontend/   React + Vite client
ingestion/  Python pipelines (EPUB parsing, OCR, embeddings)
db/         SQL schema & future migrations
dataset/    Sample source EPUBs (ignored from VCS recommendations)
processed/  JSON output generated by ingestion scripts
```

## Prerequisites

- Node.js 20+
- Python 3.12 (with [`uv`](https://github.com/astral-sh/uv))
- PostgreSQL 18.1 locally (ensure `pgvector` extension is available)
- Redis (for upcoming queue/worker integration)

## Getting Started

### Backend (Express)

```bash
cd backend
npm install
npm run dev         # starts the dev server with tsx (http://localhost:5001)
```

Build for production:

```bash
npm run build
npm run start      # runs compiled dist/server.js

# Useful scripts
npm run lint       # ESLint
npm run test       # Vitest suite
```

### Frontend (React + Vite)

```bash
cd frontend
npm install
npm run dev        # Vite dev server (defaults to http://localhost:5173)
```

### Python Ingestion

```bash
uv run python ingestion/epub/extract_epub.py dataset/Jobless\\ reincarnation -o processed -v
```

Outputs structured JSON with chapter-aligned text and image references under `processed/`.

### Database

Load the SQL schema into your local PostgreSQL instance:

```bash
psql -h localhost -U postgres -f db/schema.sql
```
